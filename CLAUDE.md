# Feature documentation workflow

- Whenever a **significant feature** is added or changed (not small fixes, typo edits,
  or config tweaks), update `README.md` to reflect it. Add or update the section that
  covers that feature.

- When **starting work on any feature**, read `README.md` first. Locate the relevant
  section before making changes.

- If `README.md` does not exist yet, create it with a sensible structure before adding
  the feature entry.
